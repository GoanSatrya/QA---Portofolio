## Project Overview
Simulated digital wallet application that allows users to store balance, transfer funds, and view transaction history.

## Testing Scope
- User authentication
- Wallet balance management
- Fund transfers
- Transaction history

## Test Strategy
- Manual testing for edge cases and financial risk scenarios
- Automation testing for stable regression flows
- API testing for transaction validation
- Performance testing for concurrent transfers

## What Is Automated (and Why)
- Login flow
- Wallet transfer happy paths
- Transaction history verification

## What Is Manual (and Why)
- Fraud scenarios
- Error handling and edge cases
- UX validation

## Tools Used
- Manual Testing: Excel, Markdown
- Automation: Playwright / Selenium
- API Testing: Postman
- Performance Testing: JMeter
